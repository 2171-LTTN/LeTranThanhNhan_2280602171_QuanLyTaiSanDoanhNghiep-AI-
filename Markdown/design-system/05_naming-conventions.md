# 🏷️ NAMING CONVENTIONS

## General Rule
Names must clearly describe purpose

---

## Java Naming

### Classes
- PascalCase
- Example: AssetService, UserController

---

### Variables
- camelCase
- Example: assetList, userId

---

### Methods
- camelCase
- Must be verbs
- Example: getAssets(), createUser()

---

## API Naming
- RESTful
- Lowercase + hyphen
- Example: /api/assets

---

## Database Naming
- Collections: lowercase
- Example: users, assets

---

## BAD Naming Examples
- data1
- temp
- x

---

## MANDATORY
- No unclear naming
